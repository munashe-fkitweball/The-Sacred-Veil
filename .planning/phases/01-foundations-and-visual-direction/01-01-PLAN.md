**Verify gate explanation (what the regex actually checks):**

The automated verify gate below greps each sketch SVG for any of these patterns and FAILS if any are found:
- `<animate` — catches `<animate>`, `<animateTransform>`, `<animateMotion>` (the SMIL animation family — sketches must be static)
- `<script` — catches inline `<script>` blocks (sketches must have no JS)
- `<image ` — catches the SVG raster `<image>` element opening tag (the trailing space avoids false-positives against any tag whose name starts with "image"; raster references violate visual-primitives discipline)
- `<linearGradient` — catches `<linearGradient>` element definitions (gradients are a faked-rendering cheat; the aesthetic is solid fills on primitives)
- `<radialGradient` — catches `<radialGradient>` element definitions (same reason as above)
- `<filter[ >]` — catches `<filter>` element definitions both with attributes (`<filter id="...">`) and bare (`<filter>`); the bracket class distinguishes the element from any tag prefixed with "filter"
- `filter=` — catches the `filter="url(#...)"` attribute reference on any element (no SVG filter chains anywhere — they're the most common way to fake blur/glow textures)

These seven patterns together enforce the visual-primitives discipline HARD RULE from PROJECT.md against every common SVG cheat: animation, scripting, raster embedding, gradient cheating, and filter-chain cheating.
